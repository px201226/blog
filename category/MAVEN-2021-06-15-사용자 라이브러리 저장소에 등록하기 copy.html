Maven 로컬 저장소에 사용자 라이브러리 등록하기
2021-06-15 > Maven

스프링 부트 기반 MSA 프로젝트를 진행 하던 중, Enum 타입이나 DTO 같은 공통 클래스를 관리하는 common 패키지를 공통모듈로 관리해야하는 일이 생겼다.

이럴 경우, 메이븐 로컬 저장소를 사용하여, 사용자 라이브러리를 모듈화 시킬 수 있다.

# Maven 프로젝트 생성

mvn archetype:generate
1

이 때, 프로젝트 버전 정보에 기본적으로 뒤에 SNAPSHOT 이 붙는데, SNAPSHOT 이 뒤에 붙으면 매번 빌드할 때 마다 버전정보가 변경되지 않아도 저장소의 최신 SNAPSHOT 파일을 가져오므로, 운영 시에는 이 부분에 주의를 해야할 것 이다.

# 라이브러리 코드 작성

나의 경우에는 라이브러리가 기능을 제공하는 것이아니라 서비스간 객체 통신을 위한 DTO 객체, 에러코드, Enum 타입 등 공통 클래스들을 넣어주기만 했다.

# 프로젝트 빌드 후 로컬 저장소 설치

mvn package    ## 프로젝트 빌드
mvn install    ## 로컬 저장소에 설치
1
2

mvn install 을 하게 되면, 사용자 폴더에 .m2 폴더 하위에 설정한 패키지 안에 설치된 jar 파일이 존재하게 된다.

# 외부 프로젝트에서 사용자 라이브러리 불러오기

<dependencies>
		<dependency>
			<groupId>com.example.myservice</groupId>
			<artifactId>common</artifactId>
			<version>0.0.1-SNAPSHOT</version>
		</dependency>
</dependencies>

외부 프로젝트의 pom.xml에 설치한 모듈의 의존성을 추가한 후, 메이븐 빌드를 하게 되면 외부 라이브러리를 Import 할 수 있다.
