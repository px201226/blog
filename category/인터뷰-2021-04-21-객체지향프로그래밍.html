Java String, StringBuffer, StringBuilder 차이
2021-04-22 > 인터뷰

String

  • Immutable(불변) 객체로 값이 변하지 않고, Thread-safy 하다.
  • String 클래스는 특수하게 new 키워드를 통해 생성하는 방법과 리터럴(literal) 방식으로 생성하는 두 가지 방법이 있다.
  • new 키워드를 생성하는 방식은 일반 객체 생성과 마찬가지로 heap 영역에 String 객체가 생성되어 String 참조 변수를 통해 참조된다.
  • "" 리터럴 방식으로 생성되는 String은 heap영역안에 있는 String constant pool 이라는 공간에 생성되어 참조된다.
  • 리터럴 방식으로 String 객체가 생성 될 경우, String constant pool 공간에서 생성할 String과 같은 값이 있는 지 탐색한다.
  • 만약, 같은 값이 String constant pool에 이미 있다면 찾은 주소값 그대로 사용하고, 그렇지 않다면 새로운 공간에 String 상수를 할당하여 사용한다.
String msg1 = "Hello";
String msg2 = "Hello";
String msg3 = new String("Hello");
String msg4 = new String("Hello");
String msg5 = new String("Hello").intern();

System.out.println(msg1 == msg2); // "Hello" == "Hello” 
System.out.println(msg3 == msg4); // new String("Hello") == new String("Hello")
System.out.println(msg1 == msg3); // "Hello" == new String("Hello")
System.out.println(msg1 == msg5); // "Hello" == new String("Hello").intern()

StringBuffer, StringBuilder

  • mutable(변경가능) 한 클래스로, 내부 문자열을 변경할 수 있다.
  • String 클래스의 경우 내부 문자열 배열이 final로 선언되어 있지만 StringBuffer, Builder의 경우에는 final로 선언되어 있지 않다.
  • StringBuffer는 내부 메소드가 synchronized 키워드를 통해 동기화하기 때문에 멀티스레드 환경에서 Thread-safy 하다.
  • StringBuilder는 일반 메소드로 선언되어 동기화를 고려하지 않기 때문에 Thread-safy 하지 않다.
  • synchronized 키워드를 사용하면 락을 걸기 때문에 StringBuffer의 경우 StringBuider 보다 성능 저하가 있다.
